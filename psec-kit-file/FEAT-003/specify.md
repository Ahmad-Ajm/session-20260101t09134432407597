# Specification: Advanced Search & Filtering (FEAT-003)

## Functional Requirements
- يجب أن يتمكن المستخدم من تصفية الفعاليات حسب:
  - المدينة (Dropdown)
  - نوع الفعالية (Dropdown)
  - القطاع (Dropdown)
  - التاريخ (Date range picker)
- التصفية متزامنة (AND logic) بين جميع الفلاتر.
- عرض عدد النتائج بشكل فوري مع كل تغيير في الفلاتر.
- يجب أن تظهر الفلاتر في أعلى صفحة قائمة الفعاليات.
- دعم كامل للغة العربية (الاتجاه، النصوص).
- واجهة متوافقة مع الشاشات الصغيرة.
- عند تغيير أي فلتر، يتم جلب النتائج من الخادم مباشرة (لا تحميل كل البيانات دفعة واحدة).
- في حال ضعف الإنترنت، تعرض رسالة مناسبة للمستخدم.

## Non-Functional Requirements
- الأداء: يجب أن تظهر النتائج خلال 2 ثانية كحد أقصى.
- الأمان: لا يسمح بحقن استعلامات (SQL Injection) أو تجاوز الصلاحيات.
- القابلية للتوسع: يمكن إضافة فلاتر جديدة مستقبلاً بسهولة.

## API Endpoints (افتراضي)
- GET /api/events?cityId=&typeId=&sectorId=&dateFrom=&dateTo=
  - جميع المعايير اختيارية.
  - يرجع قائمة الفعاليات المختصرة وعدد النتائج.

## UI Components
- FilterBarComponent (Angular)
- EventListComponent (Angular)

## Data
- المدن/الأنواع/القطاعات تُجلب من API منفصلة أو تُخزن محلياً.
