# Specification: FEAT-001 - User Registration & Login

## المتطلبات الوظيفية
1. يمكن للمستخدم إنشاء حساب جديد بإدخال (الاسم، البريد الإلكتروني، كلمة المرور، نوع الحساب: مشارك/منظم).
2. يمكن للمستخدم تسجيل الدخول بالبريد الإلكتروني وكلمة المرور.
3. يمكن للمستخدم تسجيل الدخول باستخدام Google OAuth.
4. عند التسجيل، يتم إرسال رسالة تحقق للبريد الإلكتروني.
5. يمكن للمستخدم إعادة تعيين كلمة المرور عبر البريد الإلكتروني.
6. يجب أن تظهر جميع الواجهات باللغة العربية بشكل افتراضي.
7. يجب تشفير كلمات المرور وعدم تخزينها بشكل نصي.
8. يجب أن يحدد المستخدم نوع الحساب أثناء التسجيل ولا يمكن تغييره لاحقاً من الواجهة.

## المتطلبات غير الوظيفية
- الأمان: استخدام معايير تشفير حديثة (ASP.NET Identity).
- الأداء: يجب أن تتم العمليات خلال ثوانٍ معدودة.
- التوافق: دعم كامل للجوال والمتصفحات الحديثة.
- سهولة الاستخدام: واجهات بسيطة وواضحة.

## السيناريوهات الأساسية
- تسجيل مستخدم جديد (مشارك/منظم)
- تسجيل الدخول
- إعادة تعيين كلمة المرور
- تسجيل الدخول عبر Google
- فشل التحقق من البريد الإلكتروني

## القيود
- لا يوجد دعم لتسجيل الدخول عبر Apple أو Facebook حالياً.
- لا يوجد تحقق ثنائي (2FA).
