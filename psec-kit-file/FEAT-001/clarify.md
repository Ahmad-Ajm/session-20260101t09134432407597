# Clarify: FEAT-001 - User Registration & Login

## هدف الميزة
تمكين المستخدمين من إنشاء حساب وتسجيل الدخول باستخدام البريد الإلكتروني أو وسائل التواصل الاجتماعي، مع اختيار نوع الحساب (مشارك أو منظم).

## النقاط الأساسية
- يجب دعم طريقتين للتسجيل: البريد الإلكتروني وكحد أدنى Google (OAuth).
- عند التسجيل، يحدد المستخدم نوع الحساب (مشارك/منظم).
- يجب التحقق من البريد الإلكتروني (verification email).
- يجب دعم إعادة تعيين كلمة المرور.
- يجب أن تكون واجهات التسجيل/الدخول متوافقة مع اللغة العربية.
- يجب حفظ بيانات المستخدم بشكل آمن (تشفير كلمات المرور).
- يجب توفير تجربة مستخدم سهلة وسريعة على الجوال.

## افتراضات تقنية
- سيتم استخدام ASP.NET Core Identity (ضمن ABP) لإدارة المستخدمين.
- سيتم دمج Google OAuth فقط كبداية لمواقع التواصل.
- سيتم إرسال البريد الإلكتروني عبر مزود SMTP خارجي.
- سيتم استخدام Angular لبناء واجهات المستخدم.

## حدود الميزة
- لا يشمل التحقق الثنائي (2FA) في النسخة الأولى.
- لا يشمل تسجيل الدخول عبر Apple أو Facebook حالياً.
- لا يشمل إدارة صلاحيات متقدمة (roles beyond organizer/participant).

## نقاط مفتوحة
- لا حاجة لموافقة إدارية على الحسابات الجديدة في النسخة الأولى.
- لا حاجة لربط رقم الهاتف حالياً.
