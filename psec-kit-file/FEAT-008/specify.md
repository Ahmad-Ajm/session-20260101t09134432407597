# Specification: Notifications & Alerts (FEAT-008)

## Functional Requirements
1. **Notifications Center**:
   - يظهر للمستخدم أيقونة جرس في شريط التنقل.
   - عند وجود إشعارات جديدة تظهر علامة تنبيه.
   - نافذة منبثقة أو صفحة تعرض جميع الإشعارات مرتبة زمنياً.
   - يمكن للمستخدم تمييز الإشعار كمقروء أو حذف الإشعار.

2. **Email Notifications**:
   - عند حدوث حدث يستدعي إشعار (إضافة فعالية، تذكير، تحديث، قبول/رفض)، يتم إرسال بريد إلكتروني للمستخدم.
   - يمكن للمستخدم إيقاف/تشغيل استقبال البريد من الإعدادات.

3. **Notification Triggers**:
   - إضافة فعالية جديدة في مدينة أو قطاع يهتم به المستخدم.
   - تذكير قبل موعد الفعالية بوقت محدد (24 ساعة مثلاً).
   - تحديث أو إلغاء فعالية.
   - قبول/رفض فعالية من الإدارة.

4. **Notification Management**:
   - إعدادات في لوحة المستخدم لتفعيل/تعطيل أنواع الإشعارات.
   - دعم كامل للغة العربية.

## Non-Functional Requirements
- يجب أن تصل الإشعارات خلال دقيقة واحدة من الحدث.
- يجب أن تكون الإشعارات غير متكررة لنفس الحدث.
- النظام متوافق مع الأجهزة المحمولة.
- حماية بيانات المستخدمين وعدم مشاركة بيانات التواصل مع جهات خارجية.

## Data Model (ABP/C#)
- Notification (Id, Title, Body, UserId, Type, IsRead, CreatedAt)
- UserNotificationSettings (UserId, EnableInternal, EnableEmail)

## Integration Points
- الربط مع Events Service
- الربط مع User Service
- الربط مع Email Service

## Constraints
- لا يوجد إشعارات Push في النسخة الأولى.
- جميع النصوص باللغة العربية.
