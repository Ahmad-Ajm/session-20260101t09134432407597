# Plan: Organizer Dashboard (FEAT-005)

## 1. Backend (C# ABP)
- إنشاء كيان Event مرتبط بالمنظم (OrganizerId).
- خدمات (CRUD) لإضافة/تعديل/حذف الفعاليات مع تحقق من الصلاحيات.
- تحقق من صحة البيانات (Validation) عند الإدخال.
- قيود حذف الفعالية (لا يسمح بالحذف إذا كان هناك مشاركين).
- نقاط نهاية (APIs) لإدارة الفعاليات الخاصة بالمنظم.

## 2. Frontend (Angular)
- شاشة لوحة تحكم للمنظم تعرض قائمة فعالياته.
- واجهات لإضافة وتعديل الفعالية (نماذج Forms).
- واجهة حذف مع تأكيد.
- رسائل تنبيه/خطأ واضحة.
- دعم العربية والاستجابة للأجهزة المحمولة.

## 3. Security
- تحقق من هوية المنظم (JWT/AuthGuard).
- السماح فقط للمنظمين بإدارة فعالياتهم.

## 4. Integration
- ربط مع نظام المستخدمين (User Management) للتحقق من نوع الحساب.
- ربط مع نظام الإشعارات لإعلام المنظم بنتيجة العمليات.

## 5. Testing
- اختبارات وحدات (Unit Tests) لخدمات CRUD.
- اختبارات تكامل (Integration) بين الواجهة والباكند.
