# Clarification for FEAT-002: Events Listing Interface

## What is this feature?
واجهة عرض قائمة الفعاليات الحالية والقادمة مع بيانات مختصرة مثل الاسم، التاريخ، المكان، التصنيف، والجهة المنظمة.

## Why is it needed?
- لتمكين المستخدمين من تصفح جميع الفعاليات بسهولة وسرعة.
- لتوفير نظرة عامة سريعة عن الفعاليات المتاحة.
- لتسهيل الوصول لتفاصيل الفعالية أو التسجيل.

## Who uses it?
- جميع المستخدمين (مشاركون، منظمون، زوار).

## Inputs
- بيانات الفعاليات من قاعدة البيانات (API).
- بيانات مختصرة: اسم الفعالية، التاريخ، الوقت، المكان (المدينة/العنوان)، التصنيف، الجهة المنظمة، صورة مصغرة.

## Outputs
- قائمة فعاليات بترتيب زمني (الحالية أولاً ثم القادمة).
- لكل فعالية: عرض مختصر (بطاقة/صف).
- رابط إلى صفحة تفاصيل الفعالية.

## Constraints
- دعم اللغة العربية بالكامل.
- متوافق مع الهواتف المحمولة.
- يدعم الاتصال الضعيف (تصميم بسيط).
- يجب أن يكون الأداء سريعاً حتى مع عدد كبير من الفعاليات.

## Assumptions
- يتم جلب بيانات الفعاليات عبر API (C# ABP backend).
- سيتم استخدام Angular لعرض القائمة.
- لا حاجة لعرض فعاليات منتهية في القائمة الافتراضية.

## Out of Scope
- صفحة تفاصيل الفعالية (مغطاة في FEAT-004).
- نظام الحجز/التسجيل (مغطى في ميزات أخرى).
