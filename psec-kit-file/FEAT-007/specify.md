# Specification for FEAT-007: Admin Control Panel

## Functional Requirements
1. **مراجعة الفعاليات:**
   - عرض قائمة الفعاليات بانتظار الموافقة.
   - عرض تفاصيل كل فعالية (اسم، وصف، منظم، تاريخ، مكان، تصنيف).
   - أزرار لقبول/رفض/تعديل الفعالية.
   - تأكيد قبل تنفيذ أي إجراء حساس.
2. **إحصائيات الفعاليات:**
   - عرض عدد الفعاليات المنشورة/المرفوضة/بانتظار الموافقة.
   - عرض أكثر الفعاليات زيارة (top N).
   - تصفية الإحصائيات حسب المدينة/التصنيف/الفترة الزمنية.
   - رسوم بيانية وجداول.
3. **بحث وتصفية:**
   - بحث نصي ضمن الفعاليات.
   - تصفية حسب المدينة/التصنيف/الحالة.
4. **سجل العمليات (Audit Log):**
   - تسجيل كل عملية قبول/رفض/تعديل مع اسم المدير والتاريخ.

## Non-Functional Requirements
- واجهة سهلة وسريعة الاستجابة.
- حماية الوصول عبر نظام الصلاحيات (Admins فقط).
- دعم اللغة العربية بالكامل.
- تسجيل جميع العمليات الحساسة.

## APIs
- GET /api/admin/events?status=pending
- POST /api/admin/events/{id}/approve
- POST /api/admin/events/{id}/reject
- GET /api/admin/events/statistics
- GET /api/admin/events/audit-log

## UI Components
- قائمة الفعاليات بانتظار الموافقة (جدول).
- صفحة تفاصيل الفعالية.
- أزرار الإجراءات (قبول/رفض/تعديل).
- تبويب الإحصائيات (جداول/رسوم بيانية).
- تبويب سجل العمليات.

## Security
- جميع العمليات تتطلب صلاحية Admin.
- تأكيد قبل تنفيذ أي إجراء حساس (modal confirmation).
- حماية من هجمات CSRF/XSS.

## قيود
- لا يمكن للمدير إنشاء فعالية جديدة.
- لا يمكن للمدير حذف فعالية منشورة إلا في حالات خاصة (يتم تسجيلها).

## الاعتمادية
- يعتمد على جاهزية نظام الفعاليات والصلاحيات.
