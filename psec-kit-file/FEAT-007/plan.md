# Plan for FEAT-007: Admin Control Panel

## 1. المتطلبات الأساسية
- نظام فعاليات جاهز (قواعد بيانات الفعاليات، حالة الفعالية: بانتظار الموافقة/مقبولة/مرفوضة).
- نظام صلاحيات (roles/permissions) لتحديد من يرى لوحة التحكم.
- نظام إحصائيات (داخلي أو تكامل مع Google Analytics).

## 2. واجهات المستخدم
- صفحة لوحة تحكم إدارية رئيسية.
- تبويب مراجعة الفعاليات بانتظار الموافقة.
- تبويب إحصائيات الفعاليات.

## 3. الوظائف الأساسية
- استعراض الفعاليات الجديدة مع إمكانية قبول/رفض/تعديل.
- استعراض الفعاليات المنشورة.
- عرض إحصائيات (عدد الفعاليات، الأكثر زيارة، حسب المدينة/التصنيف).
- بحث وتصفية الفعاليات.
- تسجيل العمليات الإدارية (audit log).

## 4. التقنيات المقترحة
- الواجهة الأمامية: Angular (لوحة تحكم منفصلة أو جزء من لوحة الإدارة).
- الواجهة الخلفية: C# (ASP.NET Core ضمن ABP).
- قاعدة البيانات: نفس قاعدة بيانات الفعاليات.

## 5. خطوات التنفيذ
1. إضافة صلاحيات إدارية جديدة (role: Admin).
2. بناء صفحة لوحة التحكم الإدارية (Angular route + components).
3. بناء واجهة مراجعة الفعاليات (قائمة، تفاصيل، أزرار قبول/رفض/تعديل).
4. بناء واجهة الإحصائيات (جداول/رسوم بيانية).
5. ربط العمليات مع API backend (موافقة/رفض/تعديل).
6. تسجيل العمليات (audit log).
7. اختبار الصلاحيات والوظائف.

## 6. المخاطر
- تعقيد في إدارة الصلاحيات.
- عبء على الأداء في حال كثرة البيانات (الحاجة لتصفية/ترقيم).

## 7. خطة اختبار
- اختبار صلاحيات الوصول.
- اختبار قبول/رفض الفعاليات.
- اختبار عرض الإحصائيات بدقة.
- اختبار تسجيل العمليات.
