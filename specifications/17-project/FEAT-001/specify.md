# Specification: User Registration & Login (FEAT-001)

## Functional Requirements
1. يستطيع المستخدم إنشاء حساب جديد عبر البريد الإلكتروني أو وسائل التواصل الاجتماعي (Google, Facebook).
2. عند التسجيل، يجب اختيار نوع الحساب (مشارك أو منظم).
3. التحقق من البريد الإلكتروني إلزامي عند التسجيل بالبريد الإلكتروني.
4. يمكن للمستخدم تسجيل الدخول والخروج بأمان.
5. كلمات المرور تُخزن بطريقة مشفرة (تجزئة + salt).
6. دعم كامل للغة العربية في جميع الواجهات.
7. واجهات متوافقة مع الهواتف المحمولة.

## Non-Functional Requirements
- الأمان: حماية من هجمات brute-force، حماية بيانات المستخدمين.
- الأداء: استجابة سريعة حتى في حال ضعف الإنترنت.
- التوافق: دعم جميع المتصفحات الرئيسية والأجهزة.

## User Stories
- كمستخدم جديد، أريد التسجيل بالبريد الإلكتروني أو Google/Facebook حتى أتمكن من استخدام المنصة.
- كمستخدم، أريد اختيار نوع حسابي عند التسجيل حتى أستفيد من الميزات المناسبة لدوري.
- كمنظم، أريد الدخول للوحة التحكم الخاصة بي مباشرة بعد تسجيل الدخول.

## APIs & Endpoints
- POST /api/account/register
- POST /api/account/login
- POST /api/account/social-login
- POST /api/account/logout
- POST /api/account/confirm-email

## UI Components
- صفحة التسجيل (تدعم اختيار النوع + وسائل التواصل)
- صفحة الدخول
- صفحة تأكيد البريد

## Security
- استخدام IdentityServer4 (ABP)
- حماية كلمات المرور
- Rate limiting
- حماية من CSRF وXSS
