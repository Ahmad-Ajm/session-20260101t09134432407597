# Clarify: FEAT-002 - Events Listing Interface

## Goal
إنشاء واجهة تعرض قائمة الفعاليات الحالية والقادمة مع بيانات مختصرة (اسم الفعالية، التاريخ، المكان، التصنيف، الجهة المنظمة) بشكل جذاب وسهل التصفح.

## أسئلة توضيحية
1. **ما هي الحقول الأساسية لكل فعالية؟**
   - الاسم، التاريخ، الوقت، المكان (المدينة/العنوان)، التصنيف (تعليمي، تقني، مهني...)، الجهة المنظمة، صورة مصغرة، حالة الفعالية (جارية/قادمة)، وصف مختصر.
2. **هل يجب دعم عرض الفعاليات المنتهية؟**
   - في النسخة الأولى: عرض الفعاليات الجارية والقادمة فقط.
3. **كيف يتم ترتيب الفعاليات؟**
   - الترتيب الافتراضي حسب التاريخ (الأقرب أولاً)، مع إمكانية التصفية لاحقاً.
4. **ما هي آلية جلب البيانات؟**
   - من API داخلي مبني على ABP، يعيد قائمة فعاليات مع الحقول المطلوبة.
5. **هل هناك متطلبات تصميم خاصة؟**
   - يجب أن تكون الواجهة متوافقة مع الهواتف المحمولة (responsive)، وتدعم العربية بالكامل.
6. **هل تعرض الواجهة عدد المشاركين أو معلومات إضافية؟**
   - في النسخة الأولى: لا، فقط البيانات الأساسية للفعالية.
7. **هل هناك دعم للصور؟**
   - نعم، صورة مصغرة لكل فعالية (اختيارية، تعرض صورة افتراضية في حال عدم وجود صورة).

## افتراضات
- جميع الفعاليات تملك معرف فريد.
- لا يوجد حجز مباشر من هذه الواجهة (فقط استعراض).
- يتم تحميل البيانات بشكل متدرج (pagination) لتحسين الأداء ودعم الاتصال الضعيف.

## حدود النسخة الأولى
- لا يوجد عرض للفعاليات المنتهية.
- لا يوجد دعم للبحث أو التصفية في هذه الواجهة (ميزة منفصلة FEAT-003).
- لا يوجد تفاعل مباشر (مثل الإعجاب أو الحجز) من واجهة القائمة.
