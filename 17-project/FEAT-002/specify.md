# Specification: Events Listing Interface (FEAT-002)

## المتطلبات الوظيفية
1. عرض قائمة الفعاليات الحالية والقادمة ببطاقات مختصرة.
2. لكل فعالية تظهر البيانات التالية:
   - اسم الفعالية
   - تاريخ ووقت البدء
   - المدينة/المكان
   - التصنيف (تعليمي، تقني، مهني...)
   - الجهة المنظمة
   - صورة مختصرة (اختيارية)
3. دعم اللغة العربية بالكامل.
4. تصميم متجاوب مع جميع الشاشات.
5. عرض رسالة "لا توجد فعاليات حالياً" عند عدم توفر بيانات.
6. عند الضغط على بطاقة فعالية، ينتقل المستخدم لصفحة تفاصيل الفعالية (FEAT-004).

## المتطلبات غير الوظيفية
- الأداء: تحميل سريع حتى مع وجود عدد كبير من الفعاليات.
- دعم الاتصال الضعيف (عرض بيانات نصية فقط عند الحاجة).
- أمان: لا تظهر بيانات التواصل الخاصة بالمنظمين هنا.

## واجهات التكامل
- API لجلب قائمة الفعاليات (GET /api/events)
- تكامل مع نظام التصفية والبحث (FEAT-003)

## سيناريوهات الاستخدام
1. مستخدم يفتح الصفحة الرئيسية ويرى قائمة الفعاليات.
2. مستخدم يضغط على فعالية لعرض التفاصيل.
